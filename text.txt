STATE MACHINE WORKFLOW ENGINE
Presentation Notes (Ready Script + Demo Flow)

--------------------------------------------------
1) INTRODUCTION
--------------------------------------------------
Good morning/afternoon.
Our project is a State Machine Workflow Engine.
It manages any process step-by-step using states and transitions.
The main idea is to make sure a workflow follows a valid sequence of steps.

--------------------------------------------------
2) PROBLEM STATEMENT
--------------------------------------------------
In real-world systems like food delivery, order processing, or approval systems,
workflows move through multiple steps.
Many systems are complex, so we built a simple workflow engine that:
- controls state transitions
- validates next steps
- prevents invalid flow

--------------------------------------------------
3) BASIC CONCEPTS
--------------------------------------------------
State = a step in the process
Transition = movement from one step to another
Workflow = sequence of states
Engine = logic that checks rules

Example workflow:
START -> VALIDATE -> PROCESS -> DONE

--------------------------------------------------
4) PROJECT ARCHITECTURE
--------------------------------------------------
User Request
   ->
Routes (API)
   ->
State Machine Engine (validation logic)
   ->
Response

Files:
- server.js -> starts the server
- routes/workflowRoutes.js -> handles requests
- engine/stateMachine.js -> checks valid transitions

--------------------------------------------------
5) HOW THE ENGINE WORKS
--------------------------------------------------
1. Current state is stored.
2. User asks to move to next state.
3. Engine checks if transition is allowed.
4. If valid -> state changes.
5. If invalid -> error message.

--------------------------------------------------
6) LIVE DEMO STEPS (IMPORTANT)
--------------------------------------------------
Step 1: Show current state

http://localhost:3000/state

Step 2: Move to next valid state
